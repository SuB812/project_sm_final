STEP 1. 공정위 의결서 JSON 파싱 및 지식그래프 노드 설계

프롬프트 1. 기존 수집 데이터 구조 진단

아래는 국가법령정보 Open API로 수집한 공정거래위원회 소관 법령/행정규칙

청크 파일이야 (all_chunks_v2.json, 922개).

다음을 분석해줘:

1. 각 청크의 키 구조 (page_content / metadata 내부 필드 전체)

2. 법령(law) vs 행정규칙(admrul) 분류가 source_type 필드로

정확히 나뉘어 있는지

3. 조문번호(node_id) 파싱 패턴 — "law::법령명::제N조" 형태가

일관성 있는지

4. 한 청크에 여러 조문이 묶인 경우(조문 2개 이상 감지)가 몇 건인지

5. 장·절 헤더만 있는 빈 청크가 몇 건인지

6. 법령별 청크 수 분포

프롬프트 2. 불필요 데이터 필터링 기준 설계

922개 청크 중 표시광고법과 직접 관련 없는 공정위 내부 행정규칙

(공무원 행동강령, 당직규정, 가맹사업, 기업결합 등)을 제거하고

필요한 것만 남겨야 해.

다음 기준으로 필터링 코드를 작성해줘:

[유지]

- 표시ㆍ광고의 공정화에 관한 법률 + 시행령

- 전자상거래 등에서의 소비자보호에 관한 법률 + 시행령

- 소비자기본법 + 시행령

- 약관의 규제에 관한 법률

- 과징금 관련 고시 (표시광고법 직결)

- 시정조치 운영지침

- 표시광고 심사지침류

[제거]

- 공정위 내부 운영 규정 일체

- 가맹사업, 기업결합, 하도급 관련 법령

결과를 final_chunks.json으로 저장해줘.

프롬프트 3. 특별법 표시광고 조문 수집 설계

표시광고법의 특별법에 해당하는 아래 7개 법령에서

표시·광고 관련 조문만 선택적으로 수집하고 싶어.

[대상 법령]

- 식품 등의 표시·광고에 관한 법률 (전문 수집 — 법 전체가 표시광고 전용)

- 화장품법 (제10~14조, 제36~37조)

- 의료법 (제56~57조의3, 제63조, 제67조, 제88~91조)

- 약사법 (제68조, 제68조의2, 제76조, 제93~97조)

- 의료기기법 (제24~25조, 제36조, 제52~54조)

- 건강기능식품에 관한 법률 (제17~18조, 제32조, 제44~46조)

- 방송법 (제73~75조, 제100조, 제108~109조)

국가법령정보 API (eflaw → lawService) 를 사용해서

조문키(7자리) 기준으로 파싱하되:

- 조문키 마지막 자리 0 = 장절 헤더 → 제외

- "삭제 <" 패턴 조문 → 제외

- 조문키 5~6번째 자리 = 가지번호 (02 → 제N조의2)

수집 결과를 기존 final_chunks.json에 병합 저장해줘.

프롬프트 6. 메타데이터 보강

final_all.json (774개 청크)의 메타데이터를 보강해야 해.

현재 메타데이터에는 법령유형 구분 필드가 없고,

헤더 청크(장·절 제목만 있는 청크)가 섞여 있어.

다음을 처리해줘:

[1] 법령유형 필드 추가

source 이름 기반으로 아래 규칙으로 자동 분류:

- 끝이 "시행령" → "시행령"

- 끝이 "시행규칙" → "시행규칙"

- "심사지침" 포함 → "심사지침"

- "운영지침" 포함 → "운영지침"

- "가이드라인" 포함 → "가이드라인"

- "고시" 포함 → "고시"

- "규정" 포함 → "규정"

- source_type = "특별법_표시광고" → "특별법"

- 나머지 법령 → "모법"

[2] 헤더 청크 제거

아래 조건 중 하나라도 해당하면 제거:

- page_content 길이 60자 미만이고 "제N장" 또는 "제N절" 패턴 포함

- 조문내용이 장절 제목만 있고 실질 조문 내용 없음

[3] node_id 정합성 검증 및 보정

- node_id가 "law::법령명::제N조"인데

실제 page_content가 "제N조의2" 내용인 경우 → node_id 교정

- 중복 node_id 존재 시 → 뒤에 _dup 접미사 처리

[4] 행정규칙 섹션 node_id 보강

현재 "admrul::규칙명::0", "admrul::규칙명::1" 숫자 형태인데

섹션헤더 필드(Ⅰ. Ⅱ. 등)가 있으면 이걸 node_id에 반영:

"admrul::규칙명::Ⅰ" 형태로 교체

결과를 final_all_clean.json으로 저장하고

처리 전후 청크 수 변화와 법령유형별 분포를 출력해줘.

프롬프트 7. 엣지 파일 생성 — 계층 관계 (C-1단계)

법령 간 계층 관계(모법 , 시행령 , 심사지침/고시)를

엣지 JSON 파일로 생성해야 해.

[엣지 타입 1] HAS_ENFORCEMENT (모법 → 시행령)

예시:

{

"src": "law::표시ㆍ광고의 공정화에 관한 법률",

"dst": "law::표시ㆍ광고의 공정화에 관한 법률 시행령",

"relation": "HAS_ENFORCEMENT",

"설명": "모법의 위임 규정에 따른 시행령"

}

[엣지 타입 2] HAS_GUIDELINE (모법 → 심사지침/고시)

예시:

{

"src": "law::표시ㆍ광고의 공정화에 관한 법률",

"dst": "admrul::기만적인 표시·광고 심사지침",

"relation": "HAS_GUIDELINE",

"근거조문": "제4조",

"설명": "공정위가 표시광고법 제4조에 따라 고시한 심사지침"

}

[엣지 타입 3] IS_SPECIAL_LAW (특별법 → 표시광고법)

예시:

{

"src": "special::화장품법",

"dst": "law::표시ㆍ광고의 공정화에 관한 법률",

"relation": "IS_SPECIAL_LAW",

"설명": "화장품법은 표시광고법의 특별법으로 우선 적용"

}

아래 매핑 테이블 기준으로 엣지를 생성해줘:

모법 → 시행령:

- 표시광고법 → 표시광고법 시행령

- 전자상거래소비자보호법 → 전자상거래소비자보호법 시행령

- 소비자기본법 → 소비자기본법 시행령

모법 → 심사지침/고시 (표시광고법 직결):

- 표시광고법 → 기만적인 표시·광고 심사지침 (근거: 제3조)

- 표시광고법 → 비교표시·광고에 관한 심사지침 (근거: 제3조)

- 표시광고법 → 인터넷 광고에 관한 심사지침 (근거: 제3조)

- 표시광고법 → 추천·보증 등에 관한 표시·광고 심사지침 (근거: 제3조)

- 표시광고법 → 부당한 표시·광고행위의 유형 및 기준 지정고시 (근거: 제3조)

- 표시광고법 → 표시·광고 실증에 관한 운영 (근거: 제5조)

- 표시광고법 → 중요한 표시·광고사항 고시 (근거: 제4조)

- 표시광고법 → 표시·광고의 공정화에 관한 법률 위반사업자등에 대한

과징금부과 세부기준 등에 관한 고시 (근거: 제9조)

- 약관법 → 약관심사지침

- 약관법 → 금융투자업분야 약관심사 가이드라인

- 독점규제법 → 불공정거래행위 심사지침

- 독점규제법 → 온라인쇼핑몰업자의 불공정거래행위 심사지침

- 전자상거래법 → 전자상거래 소비자보호법 위반사업자에 대한 과징금 부과기준 고시

- 전자상거래법 → 구매안전서비스에 대한 통신판매업자의 표시·광고 고시

특별법 → 표시광고법:

- 식품표시광고법, 화장품법, 의료법, 약사법,

방송법, 건강기능식품법, 의료기기법 → 표시광고법

결과를 edges_hierarchy.json으로 저장해줘.

프롬프트 8. 엣지 파일 생성 — 조문 참조 관계

각 청크의 page_content 본문에서 다른 조문을 참조하는 패턴을 추출해서

조문 간 참조 엣지를 생성해야 해.

[추출 패턴]

- "제N조제M항" → 같은 법령 내 조문 참조

- "제N조의M" → 가지 조문 참조

- "「법령명」 제N조" → 타 법령 조문 참조

- "같은 법 제N조" → 직전에 언급된 법령의 조문 참조

[엣지 타입] REFERS_TO

예시:

{

"src": "law::표시ㆍ광고의 공정화에 관한 법률::제7조(시정조치)",

"dst": "law::표시ㆍ광고의 공정화에 관한 법률::제3조(부당한 표시광고 행위의 금지)",

"relation": "REFERS_TO",

"참조문구": "제3조제1항을 위반하여",

"방향": "조건부_적용"

}

[처리 방식]

1. 정규식으로 참조 패턴 추출

2. 같은 법령 내 참조 → node_id 직접 매핑

3. 타 법령 참조 → 법령명 정규화 후 매핑

("「표시ㆍ광고의 공정화에 관한 법률」" → source 키로 탐색)

4. 매핑 실패 시 → unresolved 목록으로 별도 저장

결과를 edges_references.json으로 저장하고

매핑 성공/실패 건수를 출력해줘.

프롬프트 9. 의결서 JSON 연계

공정거래위원회 의결서 데이터(knowledge_graph.json)와

수집된 법령 청크(final_all_clean.json)를 연계해야 해.

[의결서 → 법령 조문 매핑 엣지]

엣지 타입: CITES (의결서 → 법령 조문)

예시:

{

"src": "decision::2023-공표-001::위반행위_1",

"dst": "law::표시ㆍ광고의 공정화에 관한 법률::제3조(부당한 표시광고 행위의 금지)",

"relation": "CITES",

"인용문구": "표시광고법 제3조제1항제1호",

"위반유형": "허위·과장광고"

}

[처리 방식]

1. 의결서 텍스트에서 법령 인용 패턴 추출

- "표시광고법 제N조", "법률 제N조제M항" 등

- 법령명 약칭 정규화 테이블 적용

(표시광고법 → 표시ㆍ광고의 공정화에 관한 법률)

(전자상거래법 → 전자상거래 등에서의 소비자보호에 관한 법률)
